이제 셋업이 되었으니 Multi Agent architecture를 구성해보겠습니다.
초기에는 저도 잘 몰라 각 agent를 만들고 지침 md파일을 각각 만들었습니다.
만드는 방법은 쉽습니다. antigravity의 agent manger로 실행을 합니다(agent manager화면 넣어줘)
실행을 하면 프로젝트 명이 이렇게 나오고요 여기에 +버튼으로 agent를 추가 할수 있습니다(1.agent추가.png)
저는 QER Company직원들을 총괄메니저, 작가요원, 분석요원, 개발요원으로 만들었죠.(2.agent추가결과.png)
그리고 각 요원들에게 직원들을 추가를 하였어요.(여기서 각요원들의 지침들을 너가 깔끔하게 정리해주고)
이렇게 만들어진 각 직원들의 지침들이 추가 되었습니다.(3.md파일 추가 결과.png)
자 이렇게 진행을 하면 multi agent architecture의 뼈대 작업이 모두 끝났습니다.
이렇게 해도 아무런 문제가 없고 지금 IT 유투브에서도 이렇게 진행을 하고 있을겁니다.

그러나... 최근 이러한 AGENT별로의 지침파일들이 각 Coding agent마다 표준의 필요성이 제기가 되었죠(이부분은 너가 기사를 찾아서 잘 요약해주고)
그래서 탄생한것이 AGENTS.md입니다. Coding agent들이 표준 파일을 만들고 서로 호환되게 하였다는 것입니다.
이렇게 하며 호환성의 문제 뿐만아니라 버전관리측면에서도 아주 효율적일것 같네요.
그래서 저도 당장 바꿔봤습니다. 이렇게요. (4.agent통합)
통합은 어렵지 않고 프로젝트 root 위치에 AGENTS.md파일을 만들고 각 agent들의 지침을 넣으면됩니다.
